## ✨ 新增功能

**Agent**

- **Agent 间通信**：新增 `copaw agents` 和 `copaw message` CLI 命令，支持查看 Agent 列表、向通道推送消息，以及在 Agent 之间发送请求 ([#1959](https://github.com/agentscope-ai/CoPaw/pull/1959), [#2161](https://github.com/agentscope-ai/CoPaw/pull/2161))
- **内置 QA Agent**：预置了一个 QA Agent，专门回答 CoPaw 安装和使用相关问题 ([#2189](https://github.com/agentscope-ai/CoPaw/pull/2189))
- **可配置的 LLM 自动重试**：LLM 重试行为现在可以按 Agent 在控制台设置页面中单独配置 ([#2002](https://github.com/agentscope-ai/CoPaw/pull/2002))
- **摘要优化**：当 Agent 达到最大迭代次数并进入摘要阶段时，会自动追加"本轮已结束"的提示，引导用户继续 ([#1965](https://github.com/agentscope-ai/CoPaw/pull/1965), [#2163](https://github.com/agentscope-ai/CoPaw/pull/2163))
- **配置文件自动修复**：如果 `config.json` 损坏或存在语法错误，加载时会自动尝试修复。无法修复的文件会以唯一后缀备份，CoPaw 将使用默认配置启动，不再崩溃 ([#2164](https://github.com/agentscope-ai/CoPaw/pull/2164))

**安全**

- **文件访问守卫**：新增敏感文件和目录的拒绝名单配置。开启后，Agent 的工具将被阻止读写这些路径 ([#1762](https://github.com/agentscope-ai/CoPaw/pull/1762))
- **工具守卫增强**：工具守卫功能得到增强，现在在多个工具并行执行时也能正确工作 ([#1984](https://github.com/agentscope-ai/CoPaw/pull/1984), [#1948](https://github.com/agentscope-ai/CoPaw/pull/1948))

**通道（Channels）**

- **飞书 / Lark SDK 迁移**：飞书通道已迁移至官方 `lark-oapi` SDK 并支持原生异步调用，同时新增区域选择器（飞书中国 / Lark 国际版） ([#1946](https://github.com/agentscope-ai/CoPaw/pull/1946), [#2108](https://github.com/agentscope-ai/CoPaw/pull/2108), [#2178](https://github.com/agentscope-ai/CoPaw/pull/2178))
- **小艺通道文件与图片支持**：用户现在可以通过小艺通道发送图片和文档 ([#1885](https://github.com/agentscope-ai/CoPaw/pull/1885))

**控制台与界面**

- **音视频与语音输入**：控制台聊天现已原生支持音频、视频和语音附件的发送与展示 ([#2114](https://github.com/agentscope-ai/CoPaw/pull/2114), [#2180](https://github.com/agentscope-ai/CoPaw/pull/2180))
- **流式重连**：如果在 Agent 仍在回复时刷新页面，控制台会自动重连到进行中的流式会话，并保留用户已输入的文本 ([#1931](https://github.com/agentscope-ai/CoPaw/pull/1931))
- **Web 账户管理**：开启认证后，用户可以在控制台侧边栏修改用户名和密码 ([#1989](https://github.com/agentscope-ai/CoPaw/pull/1989))
- **模型供应商搜索**：模型供应商设置页面新增搜索框，可按名称快速筛选供应商 ([#2062](https://github.com/agentscope-ai/CoPaw/pull/2062))
- **聊天滚动条**：聊天对话区域新增样式化滚动条，方便浏览长对话 ([#2083](https://github.com/agentscope-ai/CoPaw/pull/2083))

**模型供应商（Providers）**

- **多模态能力探测**：现在可以探测模型是否支持图片和视频，系统提示词中也会包含当前模型的多模态能力说明 ([#2045](https://github.com/agentscope-ai/CoPaw/pull/2045), [#2183](https://github.com/agentscope-ai/CoPaw/pull/2183))
- **Gemini 工具图片支持**：Gemini 模型现在能正确展示工具返回的图片，与 OpenAI 行为一致 ([#2127](https://github.com/agentscope-ai/CoPaw/pull/2127))

**技能与工具**

- **增强 Grep 和 Glob 搜索**：内置的 grep 和 glob 搜索工具现在会自动跳过大型目录，并支持输出大小和执行时间限制 ([#2186](https://github.com/agentscope-ai/CoPaw/pull/2186))
- **工具输出文件归入工作区**：截图、PDF、浏览器快照等工具输出文件现在保存在 Agent 的工作区目录下，而非进程工作目录 ([#2159](https://github.com/agentscope-ai/CoPaw/pull/2159))

## 🔄 变更

**核心与生命周期**

- **稳定的提示词以优化 KV Cache**：每次请求注入的环境上下文现在只包含当前日期而非完整时间戳，使提示词内容在请求间保持稳定，提升 LLM KV Cache 命中率 ([#2176](https://github.com/agentscope-ai/CoPaw/pull/2176))
- **CLI 启动加速**：CLI 命令改为延迟加载，`copaw --help` 及各子命令的启动速度显著提升 ([#2135](https://github.com/agentscope-ai/CoPaw/pull/2135))

**通道（Channels）**

- **QQ 通道重构**：简化了 QQ 通道实现，并补充了完整的单元测试 ([#1930](https://github.com/agentscope-ai/CoPaw/pull/1930))
- **QQ 和企业微信智能分块**：QQ 和企业微信上的长消息现在会在自然边界处自动拆分发送 ([#1937](https://github.com/agentscope-ai/CoPaw/pull/1937))
- **QQ WebSocket 重连**：重连次数改为可配置，并修复了清理逻辑 ([#2162](https://github.com/agentscope-ai/CoPaw/pull/2162))

**控制台与后端**

- **简化文件 I/O**：在若干非关键路径上将异步文件操作替换为同步方式，降低代码复杂度 ([#1962](https://github.com/agentscope-ai/CoPaw/pull/1962))
- **MCP 启动容错**：单个 MCP 客户端连接失败时不再阻塞整个应用启动 ([#1956](https://github.com/agentscope-ai/CoPaw/pull/1956))
- **暗色模式优化**：全面优化了暗色模式下选择控件、标题栏、多选下拉、菜单和安全设置页面的样式 ([#1920](https://github.com/agentscope-ai/CoPaw/pull/1920), [#2081](https://github.com/agentscope-ai/CoPaw/pull/2081), [#2094](https://github.com/agentscope-ai/CoPaw/pull/2094), [#2096](https://github.com/agentscope-ai/CoPaw/pull/2096))

- **macOS 桌面端构建**：修复了因 `llama-cpp-python` 依赖冲突导致 macOS 桌面端构建失败的问题 ([#1914](https://github.com/agentscope-ai/CoPaw/pull/1914))

**通道（Channels）**

**控制台与界面**

- **Agent 列表**：修复了 Agent 列表为空时的崩溃问题，并确保保存 Agent 后列表立即刷新 ([#2086](https://github.com/agentscope-ai/CoPaw/pull/2086), [#2112](https://github.com/agentscope-ai/CoPaw/pull/2112))
